CaroFront(cf) 簡介
前言
剛出社會時, 對未來矇懞懂懂
當了半年多的業務, 也當過作業員, 後來因為興趣做了兩年美編
因緣際會下接觸了 PHP, 也開始自學 Web 相關的知識
而這也是我成為 Web Developer 的起點

這一路上走來, 從後端接觸到前端, 再由前端研究到後端, 來來回回
JavaScript, PHP, ActionScript, Java, Node.js
在當中也開始接觸了一些程式架構
Java 的 Struts, Spring
Php 的 Smarty, OwnCloud, Codeigniter, Yii
JavaScript 的 jQuery, Angular, React, Bootstrap
Node.js 的 Express, Sails

而其中影響我對深的, 就是由德國人開發的 OwnCloud
在一步步熟悉之後, 漸漸的有了一個想法

何不自己開發一個 Framework, 一套屬於自己的開發工具?

於是, 當我開始接觸 Node.js 之後
一套專門為當時開發的前後端 Framework, TravelGlobal OA-System 就這樣誕生了
但因為有太多專門為公司需求而開發的項目, 那時還稱不上成熟
當我試圖把功能切分開來的時候, 卻因為公司政策的關係而停擺

其後又經過了一些磨練和波折
也接觸到現在流行的前端 Framework Angular, React
它們好不好用? 答案或許是, 但總是有太多規範和限制
而且那樣的架構已經把 js 和 html 綁定在一起了

來到現在的公司後, 就此決定開發一套專屬的前端 Framework
所以選擇了自己最熟悉的 jQuery Library 作為基底
以最小限度的限制(或者是沒有), 又可以自由擴充函式庫和功能
CaroFront 就這麼漸漸形成了

2016/08/14 by Caro.Huang
內容介紹
特色
以 jQuery 作為基礎
客製並優化功能, 作為開發基礎
動態 config
可根據不同的 url (例如 localhost, develop, production)載入相對應的設定, 減少開發複雜度
需要的基本函式庫
caro.js  : 依附於 lodash 的擴充函式庫
jQuery : 提供 Browser 基本及應用函式庫
lodash : 提供基本及應用函式庫
mobile-detect : 判斷載具函式庫, 應用於 RWC 及相容性處理
主要檔案介紹
src/_app: cf 主函式
src/config: 設定檔, 視使用情形修改  
工作分配
Lib / Serv 負責提供函式庫, 處理程式及資料邏輯
Module / Ctrl 負責 UI 單位元件
Page 負責頁面功能及動態整合
資料夾基本介紹
_compatibility: 瀏覽器相容性 程式, 如不考慮相容性可直接移除
_plugin: 外部函式庫及外掛程式
對應 gulpfile_config.js 的設定資料夾基本介紹 
coffee: coffee script 檔
css: css 檔
images: 圖檔
js: js 檔
js and coffee 分類 
ctrl: controller 模組, 類似 plugin, 用於網站客製化的功能 (e.g. 頁面右邊的選單) 
lib: 一般函式庫
module: Framework 專用外掛
page: 內頁程式 
serv: 頁面函式庫
lib / serv 函式庫介紹
差別只在於作用域不同 , 可依照使用者需求自行定義
lib 適用所有專案, serv 為專案邏輯處理
module / ctrl 模組介紹
 用法類似 jQuery plugin , 可依照使用者需求自行定義
 module 適用所有專案, ctrl 為專案客製化功能 
lib 列表
cf.ajax : $.ajax 延伸並簡化 success function; 可設定測試模式, 用來模擬 api response
cf.alert : 用 popup 模擬 alert 的功能, 避免 browser 禁用 alert
cf.cookie : 設置和讀取 cookie
cf.fb : FB API 延伸函式庫, 解決部分 API 問題, 簡化呼叫方式和防呆
cf.routeAnimate : cf.router 擴充換頁效果
cf.router : 換頁函式庫, 使用 $.ajax 切換頁面, 並利用 url hash 記錄
cf.unit : 單元函式庫, 提供一些未分類函式
module 列表
cfAddrDropdown : 台灣地址下拉選單, 可設置是否包含離島
cfCircleShow : DOM 圓盤輪播效果
cfDateDropdown : 自動化日期下拉選單, 日期下拉選單的值會依照所選的年和月改變
cfImgSwitch : 切換 <img> 圖片
cfJumpNum : 數字會亂跳, 最後顯示結果
cfListDom : 將 object-array 資料放到 DOM, 以 list 方式呈現, 例如得獎名單
cfModal : modal 視窗
cfMouseStyle : 客製化滑鼠樣式
cfPreLoad : 預載檔案, 一般用在 loading 頁面
cfRandomDrop : 隨機產生滑落物件, 例如氣泡, 水滴效果, 需搭配圖檔
cfScale : DOM 隨目標大小自動縮放, 預設為 $(window)
cfScroll : 捲軸自動滑動到 DOM 定點 的功能
cfScrollbar : 自訂捲軸樣式, 客製化外掛 mCustomScrollbar 使用方式
cfSwitchShow : 輪流顯示 DOM, 預設顯示第一個
cfSyncMove : 計算滑鼠和基準點的距離，同步移動 DOM
cfUpload : 檔案上傳
cfValidator : 表單驗證
cfWheel : 綁定滑鼠滾輪 event, 客製化外掛 jquery.mousewheel 使用方式
dom : 提供基本 fns
自動化開發管理
說明
運用 gulp 自動掃描檔案並部署網站
運用 bower 控管外掛
如無此需求可將相關檔案移除, 只使用 src 底下的檔案也可進行網站製作
特色
支援 .coffee 
在 gulp 時會自動 watch 並編譯成 js 
在自動部署時, /coffee 裡的 .coffee 會編譯成 .js 並放到 /js 底下
所以如果不使用 coffee script 請直接移除
使用 inject file 
在 gulp 時會掃描 js / css 檔, 並寫入 script, link tag 到 gulpfile_config 設定的檔案
工具
npm : Node.js Package 管理
gulp : 專案開發管理
bower : 前端外掛管理
安裝
安裝 npm 
npm install -g bower 安裝 bower
npm i 安裝開發檔案 
bower i 安裝 Framework 需要的基本函式庫
gulp 執行指令
gulp buildImg: 掃描 src/images/ 裡面的圖檔, 並輸出陣列到 /src/js/img.js
gulp buildImgWithMin: 掃描壓縮 src/images/ 裡面的圖檔, 並輸出陣列到 /src/js/img.js
gulp buildProd: 壓縮到 /dist 底下
gulp buildProdWithMap: 壓縮到 /dist 底下, 並產生 sourceMap
gulp: 自動部署, 啟動 localhost server 並監聽 coffee, js, css 檔案
gulp prod: 自動部署壓縮到 /dist 底下, 並啟動 localhost server
