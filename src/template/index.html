CaroFront(cf) 簡介
 前言
前端開發專用 FrameWork
針對目前開發項目遇到的問題做客製優化
減少開發時間及累積開發成果
 開發方式
運用 gulp 自動掃描檔案並部署網站
運用 bower 控管外掛
可根據不同的 url 載入相對應的 config
以 jQuery 搭配 gsap 實現 DOM 的動態控制
工作分配
Lib / Serv 負責提供函式庫, 處理程式及資料邏輯
Module / Ctrl 負責 UI 單位元件
Page 負責頁面功能及動態整合
開發用工具
npm
gulp
bower
使用 libraries
caro.js 
jQuery
lodash
mobile-detect
moment
gsap
支援 .coffee 
在 gulp 時會自動 watch 並編譯成 js 
在自動部署時, /coffee 裡的 .coffee 會編譯成 .js 並放到 /js 底下
所以如果不使用 coffee script 請直接移除
使用 inject file 
在 gulp 時會掃描 js / css 檔, 並寫入 script, link tag 到 src/index.html
基本安裝
安裝 npm 
npm install -g bower 安裝 bower 
npm i 安裝開發檔案 
bower i 安裝 framework 基本函式庫
gulp 執行指令
gulp buildImg : 掃描 src/images/ 裡面的圖檔, 並輸出陣列到 /src/js/img.js
gulp buildImgWithMin : 掃描壓縮 src/images/ 裡面的圖檔, 並輸出陣列到 /src/js/img.js
gulp buildBower : 將 bower 下載的外掛(bower_components), 載到 src 底下
gulp buildProd : 壓縮到 /dist 底下
gulp buildProdWithMap : 壓縮到 /dist 底下, 並產生 sourceMap
gulp : 自動部署, 並啟動 localhost server
gulp prod : 自動部署壓縮到 /dist 底下, 並啟動 localhost server
主要檔案介紹
main.html : 頁面 layout 
src/_app : cf 主函式
src/config : 設定檔, 視使用情形修改  
資料夾基本介紹 
_bower : bower 下載的檔案 
_compatibility : 瀏覽器相容性 程式
_plugin : 外掛程式
api : 後端 api   檔
coffee : coffee script 檔
css : css 檔
images : 圖檔
js : js 檔
template : 分頁內容 html  
coffee/js 分類 
ctrl : controller 模組, 類似 plugin, 用於網站客製化的功能 (e.g. 每個分頁都會有的選單) 
lib : 一般函式庫
module : framework 專用外掛
page : 內頁程式 
serv : 頁面函式庫
lib/serv 函式庫差別
差別只在於作用域不同 
lib 應用於特定功能, serv 應用於 page 的邏輯處理
module/ctrl 模組差別
 用法類似 jQuery plugin, 差別只在於作用域不同 
 module 適用所有專案, ctrl 為專案客製化功能
